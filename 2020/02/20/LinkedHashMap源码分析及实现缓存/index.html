LinkedHashMap源码分析及实现缓存

发表于 2020-02-20 | 分类于 Java ， 容器集合 |
| 字数统计: 1k 字 | 阅读时长 ≈ 4 分钟

概述

LinkedHashMap 继承自 HashMap，在 HashMap 基础上，通过维护一条双向链表，解决了 HashMap 不能随时保持遍历顺序和插入顺序一致的问题。常用于实现缓存。

Put

Map 类型的集合类是通过 put(K,V) 方法插入键值对，LinkedHashMap 本身并没有覆写父类的 put 方法，而是直接使用了父类的实现。但在 HashMap 中，put 方法插入的是 HashMap 内部类 Node 类型的节点，该类型的节点并不具备与 LinkedHashMap 内部类 Entry 及其子类型节点组成链表的能力。LinkedHashMap通过重写了HashMap的newNode()方法，实现了组成链表操作。

Remove

LinkedHashMap删除操作也是直接使用了父类的方法，那么如何在删除后从双向链表中删除对于节点呢？在HashMap的remove()方法中有afterNodeRemoval()方法，用于删除回调，LinkedHashMap重写该方法，实现链表的删除操作。

Get

默认情况下，LinkedHashMap 是按插入顺序维护链表。不过我们可以在初始化 LinkedHashMap，指定 accessOrder 参数为 true，即可让它按访问顺序维护链表。访问顺序的原理上并不复杂，当我们调用get/getOrDefault/replace等方法时，只需要将这些方法访问的节点移动到链表的尾部即可。

基于 LinkedHashMap 实现缓存

插入10个键值对后，缓存内容：
{7=49, 8=64, 9=81}

访问键值为7的节点后，缓存内容：
{8=64, 9=81, 7=49}

插入键值为1的键值对后，缓存内容：
{9=81, 7=49, 1=1}
